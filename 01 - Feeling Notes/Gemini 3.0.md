# Gemini 3.0

## 核心となる主張

GoogleのGemini 3.0は、2025年11月18日にリリースされた次世代マルチモーダルAIモデルファミリーで、推論能力・エージェント性能・長文脈処理において前世代から大幅な性能向上を達成し、開発者向けには細かい制御パラメータとストリーミング機能を提供する最先端のAIシステムである。

## 具体的な説明

### リリース情報
- **発表日**: 2025年11月18日
- **提供開始**: 発表当日から段階的展開
- **提供チャネル**: Geminiアプリ、AI Mode in Search、AI Studio（Gemini API）、Vertex AI、Gemini CLI、Google Antigravity
- **Deep Thinkモード**: 安全性検証後、数週間以内にGoogle AI Ultraサブスクライバーに提供予定

### モデルファミリー構成

#### 1. Gemini 3 Pro（フラグシップモデル）
- **入力**: テキスト、コード、画像、音声、動画、PDF
- **出力**: テキスト
- **Vertex AIモデルID**: `gemini-3-pro-preview`
- **コンテキストウィンドウ**: 入力100万トークン、出力65,536トークン

#### 2. Gemini 3 Pro Image（"Nano Banana Pro"）
- スタジオグレードの画像生成・編集モデル
- 2K/4Kオプション、高精度テキストレンダリング、マルチリファレンス合成機能
- **モデルID**: `gemini-3-pro-image-preview`
- **コンテキスト**: 入力65,536トークン、出力32,768トークン

#### 3. Gemini 3 "Deep Think"モード
- 特に難しい問題向けの強化推論モード
- 追加の安全性評価後にUltraサブスクライバー向けに展開

### 技術アーキテクチャ

- **アーキテクチャ**: Sparse Mixture-of-Experts Transformer
- **ネイティブサポート**: テキスト、視覚、音声のマルチモーダル処理
- **学習インフラ**: Google TPU + JAX + Pathways
- **知識カットオフ**: 2025年1月

### 新しい開発者向け制御機能

Gemini 2.xからの主要な改善点：

#### 1. thinking_level（推論深度制御）

**概要**:
- Gemini 3.0の内部推論量を制御する定性的パラメータ
- `thinking_budget`（2.x）の後継で、より予測可能な動作を提供

**利用可能な値**:
- **`low`**: レイテンシとコスト最小化。単純な指示実行や高スループットアプリ向け
- **`medium`**: リリース時点では未提供（近日公開予定）
- **`high`**（デフォルト）: 推論深度最大化。最初のトークン生成は遅いが、より慎重な回答

**Gemini 2.x `thinking_budget`との違い**:

| 項目        | thinking_level (3.x)    | thinking_budget (2.x)       |
| --------- | ----------------------- | --------------------------- |
| **制御タイプ** | 定性的レベル（low/medium/high） | 数値制限（0, 512, 8192等）         |
| **無効化**   | 不可（Gemini 3 Proは常に推論）   | Flash/Flash-Liteで`0`指定で無効化可 |
| **動的制御**  | レベル内でモデルが自動調整           | `-1`で動的推論（モデルが自動選択）         |
| **互換性**   | Gemini 3専用              | Gemini 2.5ファミリー専用           |
| **併用**    | 両パラメータ同時指定でエラー          | -                           |

**実用的ユースケース**:

**`low`を使う場合**:
- チャットルーティング、分類、フォーマット処理
- テンプレートベースの文章生成
- 基本的なRAG統合（最小限の推論）
- p50/p95レイテンシやスループット優先のシステム

**`high`を使う場合**:
- 複雑なコーディング/デバッグ
- マルチホップQA、数学/論理問題
- 長文書・マルチモーダル入力の分析
- 計画立案タスク
- 信頼性重視の難問（初回トークン遅延許容）

**API使用例**:
```json
{
  "generationConfig": {
    "thinkingConfig": {
      "thinking_level": "low"  // または "high"
    }
  }
}
```

**注意事項**:
- OpenAI互換レイヤーでは`reasoning_effort`が`thinking_level`にマッピング（OAI "medium" → Gemini "high"）
- Gemini 2.xのChain-of-Thoughtプロンプトハックを使っていた場合、Gemini 3の`thinking_level: "high"`でよりシンプルなプロンプトに置き換え可能

#### 2. media_resolution
視覚入力の解像度制御

#### 3. 思考シグネチャの厳格化
より信頼性の高いツール使用

#### 4. マルチモーダル関数レスポンス
関数呼び出しの拡張

#### 5. ストリーミング関数呼び出し
リアルタイム処理の改善

### ベンチマーク性能

#### 一般推論・マルチモーダル
- **LMArena Elo**: 1501（リーダーボードトップ）

- **GPQA Diamond**: 91.9%
- **MathArena Apex**: 23.4%（SOTA）
- **MMMU-Pro**: 81%
- **Video-MMMU**: 87.6%
- **SimpleQA Verified**: 72.1%（事実精度）

#### コーディング・エージェント性能
- **WebDev Arena Elo**: 1487
- **Terminal-Bench 2.0**: 54.2%
- **SWE-bench Verified**: 76.2%
- **Vending-Bench 2**: トップ性能（長期計画タスク）

#### Deep Thinkモード（社内テスト）
- **Humanity's Last Exam**: 41.0%（ツールなし）
- **GPQA Diamond**: 93.8%
- **ARC-AGI-2**: 45.1%（コード実行あり、ARC Prize検証済み）

### Gemini 2.xからの改善点

1. **性能**: 主要な推論・マルチモーダル・コーディング・事実性・長期計画ベンチマークで2.5 Proを全面的に上回る
2. **開発者体験**: レイテンシ/コスト/品質のトレードオフを最適化する実用的な制御パラメータ
3. **ツール使用**: より信頼性の高いツールオーケストレーション
4. **安全性**:
   - 迎合性の低減
   - プロンプトインジェクション耐性の向上
   - サイバー悪用への耐性強化
   - Frontier Safety Frameworkによる評価
5. **画像生成**: "Nano Banana"から"Nano Banana Pro"へのアップグレード
   - 高精度テキストレンダリング
   - ローカライゼーション機能
   - マルチリファレンス合成

### 利用方法

#### 一般ユーザー/サブスクライバー
- Geminiアプリ
- AI Mode in Search
- Deep Think（Ultraサブスクライバー向け、近日提供）

#### 開発者/企業
- **Gemini API**: AI Studio
- **Vertex AI**: `gemini-3-pro-preview`, `gemini-3-pro-image-preview`
- **Google Antigravity**: エージェント型コーディングワークフロー

## なぜ重要か

1. **マルチモーダルAIの新基準**: 100万トークンのコンテキストウィンドウにより、長文書・動画・複雑なコードベースの処理が可能
2. **エージェント性能の飛躍**: SWE-bench 76.2%、Terminal-Bench 54.2%など、実用的なコーディングタスクで高性能
3. **開発者フレンドリー**: 細かい制御パラメータにより、ユースケースに応じた最適化が可能
4. **推論能力の強化**: Deep Thinkモードによる難問解決能力の向上
5. **画像生成の進化**: テキスト含む高品質画像生成が実用レベルに
