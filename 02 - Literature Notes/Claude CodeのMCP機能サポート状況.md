**タイプ**: 📊 Analysis

## 📝 一行要約
Claude CodeはMCPの6機能中3つ（リソース・プロンプト・ツール）のみサポートし、サンプリング・ルート・エリシテーションは未実装

## 🎯 分析対象
Claude CodeにおけるMCP（Model Context Protocol）の機能実装状況

## 🔍 分析基準・枠組み
MCPプロトコル仕様の6機能を基準とし、Claude Code公式ドキュメントでの言及有無で判定

## 📈 証拠・データ

**✅ サポートされている機能（3つ）:**
1. **リソース** - @メンションで自動取得・添付
2. **プロンプト** - `/mcp__servername__promptname`形式のスラッシュコマンド
3. **ツール** - 通常のツール実行として暗黙的にサポート

**❌ サポートされていない機能（3つ）:**
4. **サンプリング** - サーバー主導のLLM呼び出し
5. **ルート** - URIやファイルシステムの境界問い合わせ
6. **エリシテーション** - ユーザーへの追加情報要求

## 🧠 解釈・推論
サポートされているのは「サーバーがクライアントに提供する機能」のみで、「クライアントがサーバーに提供する機能」は未実装。これは、Claude Codeがシンプルなクライアント・サーバー通信に特化し、複雑な双方向通信や再帰的処理を避けているためと推測される。

## 📊 結論・判断
Claude CodeのMCP実装は「基本的なツール統合」に留まり、エージェント的な自律動作（サンプリング）やセキュリティ制御（ルート）、動的対話（エリシテーション）といった高度な機能は提供されていない。実用上は十分だが、フルスペックのMCP実装ではない。

## ⚠️ 限界・反証
公式ドキュメントに記載がないだけで、実装されている可能性も否定できない。また、将来のアップデートで追加される可能性もある。

## 💡 なぜ重要か
MCPの全機能を前提とした設計をすると、Claude Codeでは動作しない可能性がある。実装する際は、サポートされている3機能のみを使用する必要がある。
